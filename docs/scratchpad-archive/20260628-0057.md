## Duplication & Smells

### Duplication
- **`pages.rs` stats-write boilerplate** ‚Äî the pattern `let mut s = state.stats.write().await; s.record_api_call("X"); drop(s);` appears 11 times across handlers (L56-59, L92-94, L165-167, L185-187, L225-227, L244-246, L259-261, L279-281, L284-286, L339-341, L379-381). Should be a helper like `state.record("X")`.
- **`pages.rs` template render error handling** ‚Äî `Html(tmpl.render().unwrap_or_else(|e| format!("Template error: {}", e)))` repeated 8 times (L84, L156, L279, L292, L302, L322, L370, L467). Extract a `render_or_error(tmpl)` helper.
- **`pages.rs` action handlers** ‚Äî `download_model`, `load_model`, `unload_model`, `delete_model`, `select_runtime`, `cancel_download` share an identical skeleton: log + stats + `match state.lms.X(...).await { Ok(msg) => Json(CommandResult{success:true,message:msg}), Err(e) => { tracing::error!; Json(CommandResult{success:false,message:e}) } }`. ~6 near-identical 15-line blocks. A `wrap_action` higher-order fn or macro would collapse ~90 LOC.
- **`api.rs` save handlers** ‚Äî `api_set_setting`, `api_chat_save`, `api_chat_clear`, `api_test_save`, `api_import` all share the `match state.db.X(...) { Ok(_) => Json(CommandResult{success:true,message:"Saved"}), Err(e) => {tracing::error!; Json(CommandResult{success:false,message:e})} }` skeleton. 5√ó ~10-line duplication.
- **`lms_client.rs` HTTP error mapping** ‚Äî `resp.json().await.map_err(|e| { tracing::error!(...); format!("Failed to parse: {}", e) })?` pattern repeated in `list_models`, `fetch_max_context`, `fetch_v0_models`, `search_huggingface`, `chat_completion`. Extract `parse_json_or_log(resp, ctx)`.
- **`lms_client.rs` `/tmp/lms-dl-{dl_id}.log` / `.pid`** path construction repeated in `download_model`, `cancel_download`, `delete_model`, `download_status`, `download_log_content` (5√ó). Should be a `dl_paths(name) -> (log_path, pid_path)` helper.
- **`models.rs::quant_vram_bytes` vs `quant_desc`** ‚Äî two parallel `match q.as_str()` arms over the same 14 quantization tokens (L249-268 and L271-290). Any new quant must be added in both places; ripe for a single lookup table `(name, bytes, desc)`.
- **`db.rs` `let conn = self.conn.lock().unwrap();`** opens 8 of 12 methods identically. Consider a private `with_conn<F, R>(f: F) -> R` helper.

### God objects
- **`LmsClient`** (lms_client.rs) ‚Äî 30 methods, 583 LOC. Mixes: (a) HTTP REST API calls, (b) SSH command execution, (c) process lifecycle (download PID tracking, orphan reaping), (d) chat completion JSON assembly, (e) host hardware probing. Clear candidate for splitting into `LmsHttpApi`, `LmsCli`, `DownloadManager`, `HostProbe`.
- **`models.rs`** ‚Äî module-level god-object: 11 unrelated structs (API DTOs, CLI parse outputs, HF API types, host info) + 6 parsers. Should split into `models/api.rs`, `models/cli_parse.rs`, `models/hf.rs`, `models/host.rs`.
- **`HfModel` impl** (models.rs L144-291) ‚Äî 8 methods, 147 LOC, mixing metadata extraction (license/languages/base_model/param_count) with VRAM recommendation logic (`recommended_quant`, `quant_vram_bytes`, `quant_desc`). VRAM logic doesn't belong on a deserialized API DTO.
